### **Topic 2.1 – Binary Numbers**

* **DAT-1.A.1** Data values can be stored in variables, lists, or constants and can be passed as input/output.
* **DAT-1.A.2** Computing devices represent data digitally; lowest-level components are bits.
* **DAT-1.A.3** A bit is shorthand for binary digit (0 or 1).
* **DAT-1.A.4** A byte is 8 bits.
* **DAT-1.A.5** Abstraction reduces complexity by hiding irrelevant details and grouping useful ones.
* **DAT-1.A.6** Bits are grouped to represent abstractions like numbers, characters, and color.
* **DAT-1.A.7** The same bit sequence may represent different data types in different contexts.
* **DAT-1.A.8** Analog data change smoothly over time (e.g., music pitch/volume, painting colors, race position).
* **DAT-1.A.9** Using digital data to approximate analog data is an example of abstraction.
* **DAT-1.A.10** Analog data can be approximated digitally using sampling (measuring values at regular intervals).
* **DAT-1.B.1** Integers are often represented with a fixed number of bits, limiting range and operations. Overflow can occur.
* **DAT-1.B.2** Some languages provide integer abstractions limited only by computer memory.
* **DAT-1.B.3** Real numbers are limited by fixed bits, leading to round-off errors. Some are stored as approximations.
* **DAT-1.C.1** Number bases (binary, decimal) are used to represent data.
* **DAT-1.C.2** Binary (base 2) uses only 0 and 1.
* **DAT-1.C.3** Decimal (base 10) uses digits 0–9.
* **DAT-1.C.4** In binary, a digit’s position determines its value (bit value × place value).
* **DAT-1.C.5** Place value is determined by base raised to the power of the position (starting from 0 on the right).

### **Topic 2.2 – Data Compression**

* **DAT-1.D.1** Compression reduces the number of bits stored or transmitted.
* **DAT-1.D.2** Fewer bits ≠ less information.
* **DAT-1.D.3** Size reduction depends on redundancy and the compression algorithm.
* **DAT-1.D.4** Lossless compression reduces bits while allowing exact reconstruction.
* **DAT-1.D.5** Lossy compression reduces bits but reconstructs only an approximation.
* **DAT-1.D.6** Lossy compression usually reduces size more than lossless.
* **DAT-1.D.7** Lossless is used when quality or exact reconstruction is most important.
* **DAT-1.D.8** Lossy is used when minimizing size or transmission time is most important.

### **Topic 2.3 – Extracting Information from Data**

* **DAT-2.A.1** Information is facts/patterns extracted from data.
* **DAT-2.A.2** Data enable identifying trends, making connections, solving problems.
* **DAT-2.A.3** Correlation in data ≠ causation; further research is needed.
* **DAT-2.A.4** Often, multiple data sources must be combined to reach conclusions.
* **DAT-2.B.1** Metadata are data about data (e.g., image’s file size or creation date).
* **DAT-2.B.2** Changing metadata doesn’t change the primary data.
* **DAT-2.B.3** Metadata help find, organize, and manage information.
* **DAT-2.B.4** Metadata can improve use of data by providing additional information.
* **DAT-2.B.5** Metadata allow structuring and organizing data.
* **DAT-2.C.1** Data processing depends on users’ and tools’ capabilities.
* **DAT-2.C.2** Challenges in data sets: cleaning, incomplete/invalid data, combining sources.
* **DAT-2.C.3** Data collected from users may be non-uniform (e.g., spelling/capitalization differences).
* **DAT-2.C.4** Cleaning data makes them uniform without changing meaning.
* **DAT-2.C.5** Bias may result from the type/source of collected data; more data doesn’t eliminate bias.
* **DAT-2.C.6** Data set size affects extractable information.
* **DAT-2.C.7** Large data sets may require parallel systems.
* **DAT-2.C.8** Scalability matters: system capacity affects data storage and processing.

### **Topic 2.4 – Using Programs with Data**

* **DAT-2.D.1** Programs can process data to acquire information.
* **DAT-2.D.2** Tables, diagrams, text, and visuals can communicate insight and knowledge.
* **DAT-2.D.3** Search tools help efficiently find information.
* **DAT-2.D.4** Data filtering systems help find information and recognize patterns.
* **DAT-2.D.5** Programs like spreadsheets organize data and find trends.
* **DAT-2.D.6** Processes for extracting/modifying data include:

  * transforming elements (e.g., doubling every list item)
  * filtering (e.g., only positive numbers, or band students)
  * combining/comparing (e.g., sum of numbers, highest GPA)
  * visualizing data (e.g., charts, graphs)
* **DAT-2.E.1** Programs process data iteratively and interactively to gain insight.
* **DAT-2.E.2** Programs can filter and clean data for knowledge gain.
* **DAT-2.E.3** Insight can come from combining, clustering, or classifying data.
* **DAT-2.E.4** Translating/transforming digital data can produce insight.
* **DAT-2.E.5** Patterns can emerge when data are transformed with programs.
